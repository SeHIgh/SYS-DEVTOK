안녕하세요 팀프로젝트 발표를 맡은 배가람입니다.
12조 발표를 시작하겠습니다.

개요)
저희 12조가 개발한 리눅스 응용 프로그램은 DevTok입니다.
DevTok이란 개발자를 뜻하는 Developer와 시계소리를 뜻하는 TickTock의 합성어로,
개발자 또는 개발을 공부하는 학생들의 공부를 돕는 타이머 앱이라는 뜻을 담고있습니다.

개발 공부를 꾸준히 하기 위해 시간을 정해놓고 공부를 하거나, 또는 문제를 일정시간 안에 푼다거나 할때
편리하게 사용할 수 있는 타이머 프로그램을 만들고자 하였습니다.

추진 배경)
프로젝트 추진 배경에 대해서 설명하겠습니다.
우리가 리눅스에 접속하는 것은 보통 개발을 위해서, 혹은 개발 공부를 위해서이죠, 따라서 
다른 어플리케이션을 실행하거나, os에 접속할 필요 없이 리눅스 안에서 공부시간을 측정하고 기록을 관리해주는 프로그램을 만들어 개발 공부를 편리하게 하고자 하였습니다.
또 리눅스 서버에 원격 접속을 통해서 여러 사용자가 사용할 수 있으니, 사용자 등록 및 접속 기능을 통해 각 사용자가 공부시간을 측정하고 기록을 관리할 수 있도록 하였습니다.

전체적인 팀 구성과 역할별 인원구성은 다음과 같습니다.

개발 환경)
운영체제로는 우분투 24.04, 통합 개발 환경으로는 visual studio code, 언어는 C를 사용하였습니다..
그 외에도 협업, 회의를 위해 github과 discord를 이용하였습니다.

사용한 주요 라이브러리들 입니다.

컴파일 방법--

다음으로는 프로젝트 코드 요약입니다.
전체적인 파일 구조를 설명드리면, 메인 동작 파일인 main.c가 있고
순위를 정렬하고 사용자 기록에 접근하는 알고리즘인 ranking.c파일과 헤더파일 ranking.h가 있습니다.
그리고 사용자들의 순위 정보를 저장하는 파일인 ranking.txt가 존재하고 users 디렉토리 안에는 사용자들의 개별 기록 파일이 저장됩니다.
프로젝트 코드는 main.c와 ranking.c를 합쳐서 총 1437줄이 나왔습니다.

구현한 주요 기능들에 대해 설명드리기 전에, 시스템 구성도를 먼저 보겠습니다.
사용자 입장에서 보기에, 사용할 수 있는 주요 메뉴들은 크게 5가지입니다. 
'사용자 등록'을 통해 사용자를 등록할 수 있고, '로그인'을 하고 접속하면 주요 기능들에 접근할 수 있습니다.
'공부하기'를 통해 공부시간을 측정하고, '기록보기'를 통해 내 기록을 보거나, 전체 사용자들의 순위를 볼 수도 있습니다.

기능에 따라 구분하면 모듈은 크게 '공부시간 측정'모듈, '사용자 파일 관리'모듈, '순위 파일 관리'모듈 3가지가 있습니다.
사용자 메뉴에서 '공부하기' 메뉴를 선택하면 공부시간을 입력받은 후, '공부시간 측정'모듈이 작동합니다. 
'공부시간 측정' 모듈은 사용자가 key를 누름에 따라 공부하기 thread와 휴식하기 thread가 번갈아가면서 작동하며 공부시간을 측정합니다.
'공부시간 측정'이 끝나면 사용자 파일과 순위파일에 정보를 저장하는데, 이때 '사용자 파일 관리' 모듈과 '순위 파일 관리' 모듈이 작동합니다.

흐름도 페이지)
전반적인 프로그램의 흐름을 나타내는 flow chart 입니다.

(기능 구현 내용 페이지로 돌아와서)
앞서 설명드린 기능들을 구현하기 위하여 사용한 주요 기법들입니다.
사용자가 키를 누를때마다 공부시간과 쉬는시간을 전환하기 위하여 thread를 사용하였고,
사용자 파일과 랭킹파일을 읽고쓰기 위하여 FILE I/O(파일입출력) 기법을 사용하였습니다.
그외에도, 터미널 상에서 gui를 구현하기 위해 terminal event기법을 사용하였고,
사용자 파일을 관리할 directory를 생성하고 읽기 위하여 File System 기법을 사용하였습니다.
